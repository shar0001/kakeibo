# 設計書：初回起動ヘルプモーダル

## 概要
アプリ初回起動時にヘルプモーダルをポップアップ表示する機能。
設定画面からも再表示できる。まだ実装はしない（提案フェーズ）。

## 設計・方針

### 初回判定
- `localStorage` の既存キー `kakeibo_settings` に `helpShown: true` を追加するだけ
- 新しいキーを増やさず、既存の設定オブジェクトに相乗り
- `false`（または未定義）なら初回 → モーダル表示後に `true` に更新

### 表示タイミング
- `startApp()` の末尾で判定し表示
- ローディングオーバーレイが消えた後なので自然に見える

### 変更が必要なファイル

| ファイル | 変更内容 |
|---------|---------|
| `index.html` | ヘルプモーダルのHTML追加（1ブロック） |
| `style.css` | モーダルのスタイル追加（既存モーダルパターンに準拠） |
| `app.js` | 表示ロジック・閉じる処理・設定画面の「使い方を見る」ボタン |

### 新規ファイル・依存関係
**なし。** Vanilla JS/CSS/HTML のみで完結。外部ライブラリ不要。

## 詳細

### モーダルに載せるコンテンツ（案）
1. 「＋」ボタンで収支を記録する
2. ホーム画面で今月の残予算を確認
3. 履歴・レポートで振り返り
4. 設定で予算・カテゴリをカスタマイズ

### 設定画面への「使い方を見る」ボタン
- リセットボタンの上あたりに追加
- `state.settings.helpShown` を `false` に戻してモーダルを再表示するだけ

### Capacitorへの影響
- なし。`npx cap copy` で Android/iOS にそのまま反映される

## 実装量の見積もり
- HTML: 20行程度
- CSS: 30行程度
- JS: 20行程度
- 合計: 小規模。1セッションで完結する作業量
